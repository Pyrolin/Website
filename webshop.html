Webshop project

Project Overview:
This project was done as part of the "Front End Web Technology" course at DTU. We as a group were only allowed to use React, Typescript, CSS and Vite. With this we should create a checkout flow for an online store.

Goal:
The goal of this project was to create the checkout flow for an online store (Front-end only).

My Role & Contributions:
I made the UX and UI for the checkout. Aditionally, i also did the routing of the site, the CSS part and a small amount of state.

Technologies Used:
HTML, CSS, Typescript, Vite.

Challenges & Solutions:
Initially, both faktura and leveringsinformation were displayed at the same time, which made the interface cluttered and confusing. This was solved by introducting a toggle for faktura. This simplified the experience and still allowing flexibility when users required different billing and delivery addresses.

Another challenge that we faced was deciding when to use React useState vs React Context. We ended up using useState for isolated, simple interactions and reserved React COntext for shared state across multiple components. This kept the codebase understandable for the project scope, and allowing us to refactor in the future.

Results:
We ended up with a webshop that worked, allowing the customer to pick and choose to remove/add products from the checkout, and to then allowing them to enter their payment options. We were very happy withn the fact that we got all the requirements done.
